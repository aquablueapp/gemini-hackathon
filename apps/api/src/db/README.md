# Drizzle ORM 数据库管理

本目录包含与数据库相关的代码，包括 schema 定义、迁移和种子数据。

## 数据库 Schema

数据库 schema 定义在 `schema` 目录中，使用 Drizzle ORM 的 TypeScript 定义。

## 初始数据插入方法

在 Drizzle ORM 中，有多种方法可以插入初始数据（种子数据）。本项目展示了三种主要方法：

### 1. 独立的种子数据脚本

文件：`seed.ts`

这种方法使用独立的脚本文件来插入初始数据。

**优点：**

- 代码组织清晰，易于维护
- 可以在不同环境中重复执行
- 可以使用 TypeScript 的类型检查
- 可以轻松添加或修改数据

**缺点：**

- 需要手动执行
- 与 schema 定义分离

**使用方法：**

```bash
pnpm db:seed
```

### 2. 在 Schema 定义中包含初始数据

文件：`schema/example-with-seed.ts`

这种方法在 schema 定义文件中包含初始数据和插入函数。

**优点：**

- 数据与 schema 定义在一起，便于管理
- 可以在应用启动时自动执行
- 可以使用 TypeScript 的类型检查

**缺点：**

- 可能导致 schema 文件变得臃肿
- 如果数据量大，可能影响应用启动时间

**使用方法：**

```typescript
import { initializeDatabase } from './db/schema/example-with-seed'

// 在应用启动时调用
await initializeDatabase()
```

### 3. 在 SQL 迁移文件中包含初始数据

文件：`drizzle/0001_initial_seed.sql`

这种方法在 SQL 迁移文件中包含 INSERT 语句。

**优点：**

- 数据随着 schema 一起迁移
- 只执行一次，不会重复插入
- 适合生产环境

**缺点：**

- 需要手动编写 SQL
- 不如 TypeScript 灵活
- 难以进行条件插入或复杂逻辑

**使用方法：**

```bash
pnpm db:migrate
```

## 推荐方法

对于不同的场景，推荐使用不同的方法：

1. **开发环境**：使用独立的种子数据脚本（方法1），便于频繁重置和测试
2. **测试环境**：使用 schema 定义中的初始数据（方法2），确保测试数据一致性
3. **生产环境**：使用 SQL 迁移文件（方法3），确保数据只插入一次

也可以组合使用这些方法，例如：

- 使用方法3插入基础数据（如语言、区域等）
- 使用方法1或方法2插入环境特定的数据

## 注意事项

- 所有方法都应使用 `onConflictDoNothing()` 或类似机制，避免重复插入
- 对于大量数据，考虑使用批量插入以提高性能
- 敏感数据（如管理员账户）应使用环境变量或配置文件，而不是硬编码
